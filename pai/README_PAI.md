## 二、把本 pai 文件夹上传到 DSW 实例
1. 在 DSW 控制台打开你的实例 →「打开」进入 JupyterLab/WebIDE。
2. 左侧文件管理器，把本地的 `pai/` 整个文件夹上传（拖拽或「上传文件夹」）。
   - 里面已含：`setup_pai.sh`、`gen_refs_pai.py`、`peach_role_v6.png`（角色锚点图）。
3. 打开一个 Terminal（菜单：File → New → Terminal）。

## 三、一键部署（约 20~40 分钟，主要是下载 ~17G 权重）
在 Terminal 里：
```bash
cd pai
bash setup_pai.sh
```
脚本会自动：检查 Python ≥3.10 → 克隆 ComfyUI → 装依赖 → 装 ComfyUI-GGUF 节点 →
下载 unet(12.34G)+text_encoder(4.68G)+vae(0.3G) → 放好 anchor → 后台启动 ComfyUI。

⚠️ 跑此脚本期间实例在运行、消耗免费计算时；跑完记得手动停止实例。
（如果卡在下权重，可 Ctrl-C 后重跑，curl `-C -` 支持断点续传。）

## 四、验证 ComfyUI 起来了
```bash
sleep 60
curl http://127.0.0.1:6889/system_stats
```
返回 JSON（含 `comfyui_version`）即成功。若报错看 `/tmp/comfy.log`。

## 五、出图
仍在 Terminal（pai 目录下）：
```bash
# 1) 先跑 test 验证整条链路（约 1~2 分钟，出一张 txt2img 桃子）
python gen_refs_pai.py test

# 2) 确认 test 出的 peach_test_*.png 长得对，再跑 full（4 张多角度，约 5~10 分钟）

```
出图结果在 `pai/out/` 目录，从 DSW 文件管理器下载回本地即可。

## 七、常见坑（已提前规避，记录备查）
1. **Python 版本**：DSW 默认可能多个 conda 环境。脚本检测 ≥3.10；若不满足，先 `conda activate <torch_env>` 再跑。
2. **torch CUDA 被改坏**：装依赖后脚本会自动校验 `torch.cuda.is_available()`，失败则重装 cu124 版。
3. **权重目录**：新版 ComfyUI 从 `models/text_encoders/` 找 CLIP，脚本同时 cp 一份到 `models/clip/` 兼容。
4. **mmproj 视觉塔**：Qwen-Image-Edit 是 instruction-based 编辑，输入图走 LoadImage→VAEEncode，不需要单独 mmproj，与 HAI 验证一致。
5. **CLIPLoaderGGUF 的 type**：用的是 `qwen_image`（HAI 0.28.0 验证通过）。若新版报错找不到该 type，按提示改成新版对应值（如 `qwen2vl`）即可。
